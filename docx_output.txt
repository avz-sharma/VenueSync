VenueSync Cloud Deployment Implementation Plan (Google Cloud Free Tier)
Objective
Deploy VenueSync as a single production-ready Google Cloud Run service where:
FastAPI serves both the API and the React frontend.
React (Vite) is built into static assets during Docker build.
Only one Cloud Run instance is deployed.
Gemini API keys are securely managed using Cloud Run environment variables.
The application scales to zero when idle to remain within the Google Cloud Free Tier.
GitHub remains the source of truth with automated deployment support.
High-Level Architecture
Browser      │      ▼Google Cloud Run      │      ▼FastAPI (Uvicorn)      │      ├──────── API Endpoints      │      ├──────── Gemini Reasoning Engine      │      └──────── Static React Build
There should be no separate frontend deployment.
The frontend must be served directly by FastAPI.

Replace the current deployment approach with a multi-stage Docker build.

Install dependencies
Build React
Produce frontend/dist
Stage 2
Python Runtime
Responsibilities:
Install Python dependencies
Copy backend
Copy frontend/dist
Launch Uvicorn
Requirements:
Small final image
Minimal attack surface
Layer caching optimized
Non-root execution where possible

Acceptance Criteria
The deployment is considered complete only when:
One Cloud Run service hosts the entire application.
React is served directly by FastAPI.
All API endpoints remain functional.
Gemini integration works using Cloud Run environment variables.
No CORS configuration is required.
The application scales to zero when idle.
Deployment succeeds directly from GitHub.
No secrets are committed.
Cold start remains acceptable.
The project remains compatible with the Google Cloud Free Tier.
Constraints
The implementation must:
Preserve existing project structure where practical.
Minimize code changes outside deployment concerns.
Avoid introducing Kubernetes, Compute Engine, or additional managed services unless required.
Keep infrastructure simple enough for hackathon demonstrations.
Produce a maintainable production configuration suitable for future scaling.
